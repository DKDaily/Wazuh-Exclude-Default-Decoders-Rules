Custom File Implementation Steps

Follow these steps to implement your custom configuration file. You can modify the file content later via the Wazuh UI or direct edit.

1 Copy the Default File to Custom Path

Copy the original file from the ruleset to your custom directory:

cp /var/ossec/ruleset/rules/default_file.xml /var/ossec/etc/rules/custom_file.xml
Note: The custom directory (`/var/ossec/etc/rules` or `decoders`) is used to protect your changes during system upgrades.

2 Set Correct Permissions and Ownership

Run these commands to set the appropriate permissions for the Wazuh manager:

chmod 660 /var/ossec/etc/rules/custom_file.xml
chown wazuh:wazuh /var/ossec/etc/rules/custom_file.xml

3 Edit ossec.conf to Exclude the Default File

Open the configuration file:

nano /var/ossec/etc/ossec.conf

Add the following exclusion line inside the <ruleset> block:

<rule_exclude>ruleset/rules/default_file.xml</rule_exclude>
Remember: The exclusion tag will be **<rule_exclude>** for rules or **<decoder_exclude>** for decoders.
Using Nano Editor: Press Ctrl+O to save, then Enter to confirm, and Ctrl+X to exit the editor.

4 Restart Wazuh Manager

Apply the changes by restarting the Wazuh Manager service:

systemctl restart wazuh-manager

5 Validate the Configuration

Run the validation command and confirm you see 'INFO: Configuration OK':

/var/ossec/bin/wazuh-modulesd -t
Tip: If errors occur, check the logs for details: tail -f /var/ossec/logs/ossec.log
